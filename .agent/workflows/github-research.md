# GitHub Research Protocol

Если пользователь просит найти "лучшее решение", "пример кода" или "библиотеку" на GitHub, используй инструмент **GitHub CLI (gh)** в терминале.
Твоя цель: МИНИМУМ токенов на вывод, МАКСИМУМ полезной информации.

## 1. Поиск репозиториев (Search)
**ЗАПРЕЩЕНО** использовать обычный \`gh search\`, так как он выводит таблицы с лишними пробелами.
Используй флаг \`--json\` для получения чистых данных.

**Команда:**
\`gh search repos "ТВОЙ ЗАПРОС" --language python --sort stars --limit 3 --json name,description,stargazersCount,url\`

*Расход: ~50-100 токенов.*


## 2. Поиск конкретного кода (Code Search)
Чтобы найти, как реализовать конкретную функцию:
\`gh search code "название_функции" --extension py\`

## 3. Быстрый анализ (Scan)
Не читай весь README сразу! Это сожжет контекст.
Прочитай только описание и первые 30 строк README, чтобы понять суть.

**Команда:**
\`gh repo view user/repo | head -n 30\`

*Расход: ~150-300 токенов (вместо 5000+ за полный файл).*

## 4. Точечное чтение кода (Sniper Mode)
Никогда не скачивай весь репозиторий.
1. Сначала посмотри список файлов:
   \`gh api repos/user/repo/contents/\`
2. Выбери нужный файл и прочитай ТОЛЬКО его:
   \`gh api repos/user/repo/contents/ПУТЬ/К/ФАЙЛУ | jq -r .content | base64 -d\`


## Summary
Алгоритм действий:
1. \`gh search ... --json\` (Нашел кандидатов)
2. \`gh repo view ... | head -n 30\` (Проверил, подходит ли)
3. \`gh api ... | base64 -d\` (Взял конкретный кусок кода)
