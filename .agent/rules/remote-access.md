# Remote Development Network Rules

Мы работаем на удаленном VPS. Локальный интерфейс (localhost) недоступен с моей машины.

## 1. Запуск серверов (Binding)
Все веб-сервисы, API и интерфейсы должны запускаться на хосте **0.0.0.0** (слушать все интерфейсы), а не `127.0.0.1` или `localhost`.
* **Правильно:** `uvicorn app:app --host 0.0.0.0 --port 8000`
* **Правильно:** `npm run dev -- --host 0.0.0.0`
* **Ошибка:** `uvicorn app:app` (по умолчанию сядет на 127.0.0.1)

## 2. Генерация ссылок
Когда ты даешь мне ссылку для открытия в браузере, всегда подставляй мой публичный IP адрес вместо localhost.
* **VPS IP:** `38.242.141.28`
* **Формат ссылки:** `http://38.242.141.28:{port}/...`

## 3. CORS и Безопасность
* Если пишешь фронтенд, учитывай, что API будет доступно по `38.242.141.28`, а не localhost. Настрой CORS соответствующим образом.
* Предупреждай меня, если открытый порт делает приложение уязвимым (например, админка без пароля), но для тестов запускай открыто.
