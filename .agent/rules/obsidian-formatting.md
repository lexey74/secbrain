# Правила форматирования контента для Obsidian

Ты должен строго придерживаться этих правил при генерации заметок, транскрибаций или отчетов, предназначенных для сохранения в базу знаний Obsidian.

## 1. YAML Frontmatter
Каждая новая заметка должна начинаться с блока метаданных. 
Пример:
---
type: note/article/transcription
project: "Second Brain"
created: {{date}}
tags: [ai, python, raw_data]
source: "telegram/youtube/web"
---

## 2. Структура заголовков
- Используй только один заголовок первого уровня (#) — это название заметки.
- Для разделов используй ## и ###. Не используй более 4 уровней вложенности.
- Оставляй пустую строку до и после каждого заголовка.

## 3. Вики-ссылки (Internal Links)
- Если в тексте упоминаются ключевые сущности (проекты, люди, технологии), оборачивай их в двойные скобки: [[Python]], [[Ollama]], [[CRM]].
- Если заметка является частью проекта, в конце добавляй ссылку на родительский проект: 
  "Родительская папка: [[Second Brain Index]]".

## 4. Оформление списков и чек-листов
- Для списков дел используй формат задач Obsidian: `- [ ] задача`.
- Для маркированных списков используй дефис `-`.

## 5. Блоки кода
- Всегда указывай язык программирования в блоке кода: ```python ... ```.
- Если код предназначен для VPS, добавляй комментарий с предполагаемым путем размещения файла.

## 6. Теги
- Используй иерархические теги через слэш: #project/second_brain, #work/crm, #status/draft.
- Не используй пробелы в тегах (используй нижнее подчеркивание).

## 7. Обработка транскрибаций
При оформлении транскрибации из Telegram-бота:
1. Выдели краткое резюме (Summary) в начале.
2. Оформи ключевые тезисы списком.
3. Сохрани оригинальные таймкоды, если они есть.
4. Добавь блок "Action Items" (что нужно сделать на основе этого текста).

## 8. Каллиграфия текста
- Используй **жирный шрифт** для акцентов на важных терминах.
- Используй > цитаты для выделения прямых мыслей из источников.
